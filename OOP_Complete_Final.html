Q1(a) - Principles of Software Engineering 2M

Answer

5 Key Principles:

  • Modularity: Break code into independent modules.
  • Abstraction: Hide complex details, show simple interface.
  • Encapsulation: Bundle data and methods, protect with access control.
  • Separation of Concerns: Each module has ONE responsibility.
  • DRY: Avoid duplicate code, reuse instead.

Q1(b) - Data Hiding vs Encapsulation 2M

Answer
  • Data Hiding: Making variables private â€“ not accessible outside the class.
  • Encapsulation: Bundling data + methods together + controlling access using getters/setters.
  • Key Difference: Data hiding is the GOAL; encapsulation is the TOOL to achieve it.

Q1(c) - Types of Inheritance 2M

Answer
  • Single: One child from one parent (A â†’ B)
  • Multilevel: Chain inheritance (A â†’ B â†’ C)
  • Hierarchical: One parent, many children (A â†’ B, C, D)
  • Multiple: One child, many parents (NOT supported for classes, only interfaces)
  • Hybrid: Mix of different types

Q1(d) - Polymorphism with Example 2M

Answer

Polymorphism = "many forms" â€“ same method name, different behavior.

class Dog extends Animal {
    @Override
    void sound() { System.out.println("Dog barks"); }
}
class Cat extends Animal {
    @Override
    void sound() { System.out.println("Cat meows"); }
}

Same method `sound()` behaves differently for each animal (runtime polymorphism).

Q1(e) - Exceptions in Java 2M

Answer

An exception is an error during program execution. Java provides try-catch-finally to handle gracefully.

  • Examples: `NullPointerException`, `IOException`, `ArithmeticException`
  • Using exception handling, program doesn't crash suddenly.
  • Can throw, catch, and handle exceptions to recover from errors.

Q2(a) - Class vs Object 5M

Full Answer

Class: Blueprint/template defining properties and methods. Logical definition, not real data.

Object: Real instance created in memory using `new` keyword. Has actual values for fields.

class Car {           // Class (blueprint)
    String model;
    void start() { }
}

Car car1 = new Car(); // car1 is an object
car1.model = "Honda"; // This object has actual data
The class `Car` is the blueprint. `car1` is the actual object in memory with specific values.

Q2(b) - Static Keyword 5M

Full Answer

Static members belong to the CLASS, not to individual objects. One copy shared by all.

class Counter {
    static int count = 0;  // Shared by all objects

    Counter() { count++; }

    static void show() {   // Can be called without object
        System.out.println("Count: " + count);
    }
}

Counter c1 = new Counter(); // count = 1
Counter c2 = new Counter(); // count = 2
Counter.show();              // Output: Count: 2
Static `count` is shared. All objects increment the same variable. Static method `show()` called via class name.

Q4(a) - Polymorphism and Types 5M

Full Answer

Type 1 - Compile-Time (Method Overloading): Same name, different parameters.

class Math {
    public int add(int a, int b) { return a + b; }
    public double add(double a, double b) { return a + b; }
}

Type 2 - Runtime (Method Overriding): Same name in parent and child, different implementations.

class Animal { void sound() { } }
class Dog extends Animal {
    @Override void sound() { System.out.println("Bark"); }
}

Q5(b) - throw vs throws 5M

Full Answer

throw: Inside method body to explicitly throw exception.

void validate(int age) {
    if (age < 18) throw new IllegalArgumentException("Age < 18");
}

throws: In method signature to declare method might throw exception.

void readFile() throws IOException {
    FileReader fr = new FileReader("data.txt");
}

Q6(a) - Checked and Unchecked Exceptions 5M

Program

Checked Exception (IOException): Must be caught or declared.

try {
    FileReader fr = new FileReader("file.txt");
} catch (IOException e) {
    System.out.println("File error: " + e);
}

Unchecked Exception (ArithmeticException): Not checked by compiler.

try {
    int result = 10 / 0;
} catch (ArithmeticException e) {
    System.out.println("Error: " + e);
}

Q11(a) - Password Validation 10M

Complete Program 
import java.util.Scanner;

class AuthenticationFailureException extends Exception {
    public AuthenticationFailureException(String message) {
        super(message);
    }
}

class PasswordValidator {
    private static final String CORRECT_PASSWORD = "SecurePass123";
    private static final int MAX_ATTEMPTS = 3;

    public void validate(String password) throws AuthenticationFailureException {
        if (password == null || password.isEmpty()) {
            throw new AuthenticationFailureException("Password cannot be empty");
        }
        if (!password.equals(CORRECT_PASSWORD)) {
            throw new AuthenticationFailureException("Invalid password");
        }
    }

    public void loginWithAttempts() {
        Scanner sc = new Scanner(System.in);
        for (int i = 1; i <= MAX_ATTEMPTS; i++) {
            try {
                System.out.print("Enter password (Attempt " + i + "/" + MAX_ATTEMPTS + "): ");
                validate(sc.nextLine());
                System.out.println("Login successful!");
                return;
            } catch (AuthenticationFailureException e) {
                System.out.println("Error: " + e.getMessage());
                if (i == MAX_ATTEMPTS) System.out.println("Account locked!");
            }
        }
    }
}

public class Main {
    public static void main(String[] args) {
        new PasswordValidator().loginWithAttempts();
    }
}

public class FinallyDemo {
    static void caseNoException() {
        System.out.println("=== Case 1: No Exception ===");
        try {
            System.out.println("Try block");
            int x = 10 / 2;
        } catch (ArithmeticException e) {
            System.out.println("Catch block");
        } finally {
            System.out.println("Finally block (ALWAYS EXECUTES)\n");
        }
    }

    static void caseException() {
        System.out.println("=== Case 2: Exception Occurs ===");
        try {
            System.out.println("Try block");
            int x = 10 / 0;  // Exception
        } catch (ArithmeticException e) {
            System.out.println("Catch block: " + e);
        } finally {
            System.out.println("Finally block (ALWAYS EXECUTES)\n");
        }
    }

    public static void main(String[] args) {
        caseNoException();
        caseException();
        System.out.println("Program completed");
    }
}

Q12(a) - Author and Publication Polymorphism 10M

Complete Program 
class Author {
    protected String name;
    public Author(String name) { this.name = name; }
    void display() { System.out.println("Author: " + name); }
}

class BookPublication extends Author {
    String bookTitle;
    int year;
    public BookPublication(String name, String title, int y) {
        super(name);
        bookTitle = title;
        year = y;
    }
    void publish() {
        display();
        System.out.println("Book: " + bookTitle + " (" + year + ")");
    }
}

class PaperPublication extends Author {
    String paperTitle;
    String journal;
    public PaperPublication(String name, String title, String j) {
        super(name);
        paperTitle = title;
        journal = j;
    }
    void publish() {
        display();
        System.out.println("Paper: " + paperTitle + " in " + journal);
    }
}

public class Main {
    public static void main(String[] args) {
        Author[] pubs = {
            new BookPublication("Haruki", "Norwegian Wood", 1987),
            new PaperPublication("Einstein", "Relativity", "Annalen")
        };

        for (int i = 0; i < pubs.length; i++) {
            if (pubs[i] instanceof BookPublication)
                ((BookPublication) pubs[i]).publish();
            else if (pubs[i] instanceof PaperPublication)
                ((PaperPublication) pubs[i]).publish();
        }
    }
}

ðŸŽ¯ Key OOP Concepts

IS-A vs HAS-A Relationship

IS-A (Inheritance): Dog IS-A Animal (extends keyword)

HAS-A (Composition): Car HAS-A Engine (instance variable)

Constructor Concept

Special method called when object is created. Initializes object with initial values.

  • Same name as class
  • No return type
  • Called automatically with `new` keyword
  • Can be overloaded

this vs super Keywords

this: Refers to current object. Access current class fields/methods.

super: Refers to parent class. Access parent fields/methods.
